# Compliance 90-Day Readiness Plan

This plan is designed for BuildSync beta readiness and execution toward ISO 27001, SOC 2 Type II, and GDPR readiness.

## Program Outcomes by Day 90

- Unified control matrix with owners, mappings, and evidence links
- Readiness-level implementation for core ISO/SOC controls
- GDPR operational readiness (ROPA, DSR process, transfer safeguards)
- External audit readiness package prepared

## Phase 1: Days 1-30 (Foundation)

### Milestones

1. Program kickoff complete and scope approved
2. Control ownership assigned for all in-scope domains
3. Baseline risk assessment completed
4. Gap assessment complete for ISO 27001, SOC 2, GDPR

### Deliverables

- In-scope system boundary and data flow diagrams
- Version 1 control matrix populated with initial status
- Evidence repository structure created
- Policy set draft: access, change, incident, backup, vendor, privacy
- Subprocessor inventory and contract review initiated

### Exit Criteria

- 100% controls have an owner
- Top 10 high-risk gaps have remediation tickets
- Executive sign-off on scope and timeline

## Phase 2: Days 31-60 (Implementation)

### Milestones

1. Core technical controls implemented and tested
2. Operational processes moved from ad-hoc to repeatable
3. Privacy operations running in production workflow
4. First internal control effectiveness checks completed

### Deliverables

- MFA and privileged access controls enforced
- CI security scans and change-approval evidence automated
- Centralized logging and alerting for critical actions
- Incident response tabletop completed and remediations logged
- DSR intake workflow and SLA dashboard live
- Retention/deletion controls documented and tested

### Exit Criteria

- At least 70% controls at Implemented or In Progress with evidence
- No unassigned High severity remediation items
- Signed internal readiness checkpoint report

## Phase 3: Days 61-90 (Validation & Audit Prep)

### Milestones

1. Internal audit and management review completed
2. Readiness package prepared for external auditor
3. SOC 2 observation window start criteria met
4. Beta trust statement aligned with validated posture

### Deliverables

- Internal audit report with findings and corrective actions
- Management review minutes and accepted risk exceptions
- SOC 2 evidence binder starter set (security criteria)
- ISO readiness packet (SoA, risk treatment plan, control evidence index)
- GDPR readiness packet (ROPA, DPIAs, DSR logs, transfer mechanism docs)
- Public compliance statement updated to reflect actual status

### Exit Criteria

- 90% of in-scope controls evidence-linked and tested
- 0 High-risk controls without approved mitigation timeline
- External auditor scoping call completed

## Governance Cadence

- Weekly: Compliance stand-up (30 minutes)
- Biweekly: Remediation review with engineering and security owners
- Monthly: Executive steering checkpoint
- Quarterly: Formal management review and risk acceptance

## Beta Launch Compliance Gate

Before launching beta broadly:

1. Control matrix and evidence checklist are actively maintained.
2. Trust statements on product pages exactly match validated status.
3. Incident communication templates are approved.
4. DSR and privacy escalation paths are tested end-to-end.
5. High-risk gaps have funded owners and delivery dates.
